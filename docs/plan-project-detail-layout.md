# План: новый шаблон детальной страницы проекта

## Цель
- Страница на весь экран, **без скролла всей страницы** (ни вертикального, ни горизонтального).
- **Две колонки**: слева — скролл только по картинкам; справа — фиксированный блок с заголовком и текстом.
- При скролле левой колонки **правая часть обновляется** (заголовок и текст текущей сцены/блока).

---

## 1. Разметка и запрет скролла страницы

**Файл:** `app/projects/[slug]/page.tsx`

- Обернуть контент в layout **на весь экран**: `h-screen w-screen overflow-hidden`, без `min-h-screen` и без скролла у `main`.
- Внутри — две колонки (например, grid или flex):
  - **Левая** — скроллируемый контейнер только для картинок (`overflow-y-auto`, `flex-1` или заданная ширина).
  - **Правая** — фиксированный блок (через `position: sticky` и `top-0` или фиксированная ширина + `overflow-hidden`), скролла нет.
- Кнопку «Проекты» оставить в общем шапке или в правом блоке, чтобы она не уезжала при скролле (например, в правой колонке сверху).

**Итог:** ни body, ни main не скроллятся; скролл только у левой колонки.

---

## 2. Левая колонка — только скроллируемые картинки

**Компоненты:** новая обёртка или доработка `SceneGallery` / сцен.

- Левая колонка — один контейнер с `overflow-y-auto` и секциями по одной картинке на каждую сцену (и при желании один блок сверху под обложку проекта).
- Каждая секция:
  - занимает полную ширину колонки и достаточную высоту (например, `min-h-screen` или `min-h-[80vh]`), чтобы при скролле «активной» становилась одна секция;
  - внутри — одна картинка (Next.js `Image`) на всю секцию.
- Данные: массив сцен из `project.scenes` (уже приходят с страницы) + при необходимости `project.thumbnail` для первого блока.
- Не обязательно сохранять текущий вид галереи с превью и кнопками; фокус — «вертикальный скролл по большим картинкам».

**Итог:** слева только вертикальный скролл по картинкам, без скролла остальной страницы.

---

## 3. Правая колонка — фиксированный блок (заголовок и текст)

**Файл:** страница или новый клиентский компонент (например, `ProjectDetailLayout`).

- Правая колонка не скроллится и не зависит от скролла по высоте (контент может обрезаться или быть коротким).
- Внутри блока:
  - **Общие данные проекта:** заголовок проекта, описание проекта, теги (как сейчас).
  - **Текущая сцена:** заголовок и описание той сцены, которая «в зоне видимости» в левой колонке (см. п. 4).
- Оформление оставить в текущем стиле (шрифты, цвета, отступы), без лишних элементов.

**Итог:** справа всегда виден один и тот же блок; меняется только блок «текущая сцена» (заголовок + текст).

---

## 4. Связь скролла слева и контента справа (обновление заголовка и текста)

**Реализация:** scroll-spy по левой колонке.

- В клиентском компоненте, который рендерит левую колонку (или общий layout страницы):
  - использовать **Intersection Observer** по секциям с картинками (каждая секция — один `div` с ref);
  - считать «активной» сцену, чья секция больше всего попадает во viewport (например, порог 0.5) или первая видимая сверху.
- Хранить в state `activeSceneId` (или индекс) и передавать в правую колонку.
- Правая колонка по `activeSceneId` показывает:
  - заголовок и описание этой сцены;
  - если активна «обложка проекта» (первый блок) — показывать заголовок/описание проекта, без сцены.
- При необходимости — лёгкая анимация смены текста (opacity/transition), без изменения функциональности.

**Итог:** при скролле картинок слева заголовок и текст справа обновляются под текущий блок.

---

## 5. Разделение серверной и клиентской частей

- **Серверная часть** (`app/projects/[slug]/page.tsx`): как сейчас — `getProjectBySlug(slug)`, проверка `notFound()`, передача `project` в клиентский layout/компонент.
- **Клиентский компонент** (например, `ProjectDetailView` или `ProjectDetailLayout`): принимает `project`, рендерит:
  - левую колонку (список сцен + опционально обложка) с ref’ами для Intersection Observer;
  - правую колонку с проектом и текущей сценой;
  - при необходимости — кнопку «Проекты» в шапке правого блока.
- Страница остаётся серверным компонентом; тяжёлая интерактивность и scroll-spy — только в клиентском компоненте.

**Итог:** данные и роутинг не меняются, меняется только способ отображения и реакция на скролл.

---

## 6. Обновление шаблона в Sanity

Чтобы контент-модель и Studio соответствовали новому шаблону детальной страницы.

### 6.1. Схема документов (schema)

**Файл:** `sanity/schemaTypes/index.ts`

- **Текущих полей достаточно** для нового макета:
  - **Project:** `title`, `description`, `thumbnail`, `scenes` — заголовок и описание идут в правый блок, обложка и сцены — в левый скролл.
  - **Scene:** `title`, `description`, `image`, `order` — при скролле слева справа показываются именно title и description сцены.

- **Опционально (по желанию):**
  - В **Project** добавить поле «Подзаголовок» / «Краткий текст» для первого экрана (если нужно отличать от основного описания). Если не нужно — не трогать.
  - В **Scene** при необходимости добавить поле «Подзаголовок» (короткая строка над описанием). По умолчанию можно обойтись без изменений схемы.

- **Обязательно:** проверить, что у сцен заполняется **Order** — от него зависит порядок блоков в скролле. В GROQ уже есть сортировка `order(order asc)`.

**Итог:** схему можно не менять или добавить 1–2 опциональных поля; текущей модели хватает для «слева картинки, справа заголовок и текст по скроллу».

### 6.2. Подсказки в Studio (labels и descriptions)

- В **Project** для полей задать понятные подсказки под новый шаблон:
  - **Title** — «Заголовок проекта. Показывается в правой колонке на детальной странице.»
  - **Description** — «Описание проекта. Отображается в правом блоке, когда в зоне видимости первая картинка (обложка).»
  - **Thumbnail** — «Обложка проекта. Первая картинка в левой колонке на детальной странице.»
  - **Scenes** — «Блоки с картинками в левой колонке. При скролле справа подставляются заголовок и описание выбранной сцены.»

- В **Scene**:
  - **Title** — «Заголовок сцены. Показывается справа при прокрутке до этой картинки.»
  - **Description** — «Текст сцены. Показывается справа при прокрутке до этой картинки.»
  - **Image** — «Картинка блока в левой колонке.»
  - **Order** — «Порядок блока на детальной странице (меньше число — выше).»

Реализация: в `defineField` добавить `description: '...'` (и при желании русифицировать `title`).

**Итог:** редакторам понятно, что куда попадает на новом шаблоне.

### 6.3. Структура Studio (structure tool)

**Файл:** `sanity.config.ts` или отдельный structure-файл.

- Добавить кастомную структуру списка документов (через `structureTool({ structure })` или `defaultDocumentNode`), чтобы:
  - в боковой панели были пункты **«Проекты»** и **«Сцены»** с иконками и, при желании, русскими названиями;
  - при открытии проекта поля были сгруппированы логично: сначала «Детальная страница» (title, description, thumbnail, scenes), потом при необходимости «Мета» (slug, date, tags).

- Опционально: для типа **Project** задать порядок полей в форме так, чтобы он совпадал с порядком на сайте (обложка → сцены → остальное).

**Итог:** в Sanity видно, что контент привязан к новому шаблону детальной страницы, структура Studio этому не мешает.

### 6.4. Проверка после изменений

- Убедиться, что в `sanity/queries.ts` по-прежнему запрашиваются все нужные поля: `title`, `description`, `thumbnail`, `scenes` с вложенными `title`, `description`, `image`, `order`. При добавлении новых полей (подзаголовок и т.п.) — добавить их в GROQ и в типы TypeScript.
- После деплоя Studio переопубликовать проект/сцены не требуется — меняются только подсказки и структура; данные остаются совместимыми.

**Итог:** обновление шаблона в Sanity сводится к подсказкам, опционально — к 1–2 полям и кастомной структуре Studio; текущая схема поддерживает новый макет.

## 7. Стили и крайние случаи

- Сохранить текущую палитру и типографику (фон `#00060a`, акценты, шрифты).
- Адаптив: на узких экранах можно временно оставить один колонку (например, сверху фиксированный блок, снизу скролл картинок) или упрощённый вариант без двух колонок; в плане приоритет — десктоп: две колонки, без скролла страницы.
- Если сцен нет: показывать только обложку проекта слева и блок проекта справа, без блока «текущая сцена».
- Если сцена одна: scroll-spy всё равно работает, справа один вариант заголовка/текста.
